Работа с наборами данных (dataset)

Набор данных представляет из себя таблицу с полями заданного типа, в которой хранятся записи с данными по этим полям.
 
Правила наборов данных:
  • Для каждого набора данных должно указываться соединение, через которое будет вестись работа. Не требуется обязательного указания соединения для временных источников данных (embeddedTable и csvTemp), а также для файловых источников excel, Xml, Json и Yaml, если у них в имени файла явно задан путь к нему;
  • Соединение должно быть того же типа, что и набор данных. В зависимости от типа объект dataset будет иметь разный типовой набор атрибутов и методов;
  • У каждого dataset могут быть явно описаны поля его структуры field или же получены из метаданных источника, если это поддерживается.
 
Поддерживаемые наборы данных:
Набор данных
Соединение
Особенности
csv
csvConnection
  • Чтение и запись текстовых файлов csv в любой кодировке;
  • Поддерживаются форматы traditional unix и rfc4180 windows;
  • Поддерживается чтение полей из заголовка файла;
  • Можно включить ведение файлов схем, которые будут созданы при записи и использованы для чтения структуры файла при его чтении;
  • Есть поддержка констрейнтов при чтении и записи файлов;
  • Есть работа с упакованными gz файлами;
  • Возможна разбивка при записи файлов по заданному размеру или условию и чтение порций разбитого файла как одного набора данных;
  • Для записи в один файл из множества потоков поддерживается опция append.
csvTemp
csvTempConnection
  • Csv файл, который создается во временном директории ОС с уникальным именем и автоматически удаляется при завершении работы приложения.
db2Table
db2Connection
  • Чтение и запись таблиц РСУБД IBM DB2, выполнение SQL запросов.
embeddedTable
embeddedConnection
  • Чтение и запись таблиц встроенной inmemory РСУБД H2, выполнение SQL запросов.
excel
excelConnection
  • Чтение листов файлов Excel;
  • Возможность чтения списка полей из первого ряда листа и автоопределение типов полей.
firebirdTable
firebirdConnection
  • Чтение и запись таблиц РСУБД Firebird 3, выполнение SQL запросов.
h2Table
h2Connection
  • Чтение и запись таблиц РСУБД H2 database, выполнение SQL запросов;
  • Поддержка пакетной загрузки csv файлов в таблицы;
  • Поддержка работы в режиме inmemory, обычного OLTP или гибридной схемы работы;
  • Возможность при создании таблиц задать им нетранзакционное поведение и хранение в памяти;
  • Поддержка подключения собственных udf функций, разработанных на Java/Groovy.
hiveTable
hiveConnection
  • Чтение и запись таблиц Hadoop Hive, выполнение SQL запросов;
  • Возможность при создании таблиц задать им формат хранения, путь хранения, сортировку и сегментацию;
  • Поддержка пакетной загрузки csv файлов в таблицы.
impalaTable
impalaConnection
  • Чтение и запись таблиц Cloudera Impala, выполнение SQL запросов;
  • Возможность указать опции записи: компрессию блоков, перезапись данных таблицы;
  • Возможность при создании таблиц задать им формат хранения, путь хранения, сортировку и сегментацию;
  • Поддержка пакетной загрузки csv файлов в таблицы.
json
jsonConnection
  • Чтение json файлов в любой кодировке;
  • Есть работа с упакованными gz файлами;
  • Возможно указать путь доступа к нодам схемы для корректного чтения полей.
mssqlTable
mssqlConnection
  • Чтение и запись таблиц РСУБД MS SQLServer, выполнение SQL запросов;
  • При чтении записей можно указать хинт "with".
mysqlTable
mysqlConnection
  • Чтение и запись таблиц сервера РСУБД MySql, выполнение SQL запросов.
netezzaTable
netezzaConnection
  • Чтение и запись таблиц сервера РСУБД IBM Netezza, выполнение SQL запросов.
netsuiteTable
netsuiteConnection
  • Чтение данных приложений NetSuite, выполнение SQL запросов.
oracleTable
oracleConnection
  • Чтение и запись таблиц сервера РСУБД Oracle, выполнение SQL запросов;
  • При чтении записей можно задать SCN и партицию, а так же указать собственные хинты для SELECT.
postgresqlTable
postgresqlConnection
  • Чтение и запись таблиц сервера РСУБД PostgreSQL, выполнение SQL запросов.
query / view
JdbcConnection
  • Чтение представлений и SQL запросов с любого JDBC совместимого сервера РСУБД.
salesforce / salesforceQuery
salesforceConnection
  • Чтение данных приложения SalesForce, выполнение запросов к данным.
verticaTable
verticaConnection
  • Чтение и запись таблиц РСУБД Micro Focus Vertica, выполнение SQL запросов;
  • Возможность при создании таблиц задать им партиционирование, сортировку и сегментацию;
  • Возможность для чтения записей из таблиц задать хинт "label" и указать процент сэмплирования TABLE SAMPLE;
  • Возможность при записи в таблиц задать хинт "label" и указать через direct, куда идет запись: в WOS или ROS;
  • Поддержка пакетной загрузки csv файлов в таблицы через штатный парсер и дополнительные парсеры Vertica с поддержкой всех их параметров.
xero
xeroConnection
  • Чтение данных приложения Xero;
  • На текущий момент поддерживается только 1 версия API доступа к облаку Xero.
xml
xmlConnection
  • Чтение json файлов в любой кодировке;
  • Есть работа с упакованными gz файлами;
  • Возможно получение значений атрибутов файла XML;
  • Возможно указать путь доступа к нодам схемы для корректного чтения полей.
 
Пример описания РСУБД таблицы Vertica:
verticaTable(‘ver:table1’, true) {
  useConnection verticaConnection(‘ver:con’)
  schemaName = 'demo'
  tableName = 'table1'
 
  createOpts {
    orderBy = ['dt', 'id']
    segmentedBy = 'hash(id) all nodes'
    partitionBy = 'dt::date'
    ifNotExists = true
  }
 
  dropOpts { ifExists = true }
  readOpts { label = 'my query hint' }
 
  writeOpts { direct = 'AUTO' }
 
  bulkLoadOpts {
    compressed = 'GZIP'
    loadMethod = 'DIRECT'
    enforceLenth = true
  }
 
  field(‘id’) { type = integerFieldType; isKey = true }
  field(‘name’) { length = 150; isNull = false }
  field(‘dt’) { type = datetimeFieldType; isNull = false }
}
 
Пример описания CSV файла:
csv(‘csv:file1’, true) {
  useConnection csvConnection(‘csv:con’)
  fileName = 'file1'
  fieldDelimiter = ','
  rowDelimiter = '\r\n'
  encodePage = 'utf-8'
  header = true
  locale = 'en-EN'
  formatDate = ‘yyyy-MM-dd’
 
  field(‘num’) { type = integerFieldType }
  field(‘date’) { type = dateFieldType }
}
 
Пример описания XML файла:
xml(‘xml:file1’, true) {
  // Файл находится в ресурсном файле проекта
  fileName = 'resource:/file1.xml'
 
  // По умолчанию значения полей берутся из нод элементов XML
  defaultAccessMethod = DEFAULT_NODE_ACCESS
 
  // Список записей берется по пути XML “group:data”->”group.list”
  rootNode = ‘group:data.group.list’
 
  field(‘num’) { type = integerFieldType }
  // У поля свой формат парсинга даты
  field(‘date’) { type = dateFieldType; format = ‘dd-MM-yyyy’ }
  // Поле берется как атрибут тэга ноды
  field(‘type’) { type = stringFieldType; alias = ‘@type’ }
}
 
Пример описания данных из SalesForce объекта:
salesforce(‘csv:file1’, true) {
  useConnection salesforceConnection(‘sf:con’)
  sfObjectName = ‘Account’
}
